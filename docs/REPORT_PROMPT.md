# 给网页版 ChatGPT 的报告生成提示词

你现在需要基于当前仓库内容，帮我撰写《操作系统课程设计报告》正文草稿。请严格遵守以下要求：

## 角色与口吻

- 以我的第一人称来写，使用“我”。
- 不要写成第三人称说明文。
- 不要代替我填写封面信息。

## 内容范围

- 依据仓库中的源代码、`RUN.md`、`docs/REPORT_QA.md`、`docs/SCREENSHOT_PLAN.md` 和指导书 PDF 生成报告。
- 报告中必须体现指导书中的提问与回答。
- 每个实验都要写出“我做了什么、运行结果说明了什么、我如何分析”。

## 结构要求

- 建议按实验顺序写：A1-A5，B1-B9。
- 每个实验至少包含：
  - 实验目的
  - 实验内容
  - 数据结构或算法思想
  - 关键代码说明
  - 运行结果与分析
  - 指导书问题回答
  - 小结

## 图片与流程图

- 对所有需要截图的位置，先保留占位符，不要虚构截图内容。
- 占位符格式示例：`[截图占位：B6 help 命令输出]`
- 如果需要流程图，请给出适合 draw.io 绘制的简洁流程说明。
- 流程图风格要求：白底、单色、简单、规整。

## 代码展示要求

- 不要大段粘贴全部代码。
- 只摘录关键代码。
- 代码建议以 Word 表格形式呈现。
- 需要高亮关键字和关键 API。

## 排版提醒

- 正文字号按五号处理
- 代码字体使用 `JetBrainsMono NFM Medium`
- 内容宁可详细，不要过于简略

## 分析要求

- 不能只写“程序运行成功”。
- 必须解释输出含义。
- 若涉及教材数据或教材图示，默认以教材为准进行分析。

## 额外提醒

- 结尾的小结与心得体会要写得像学生本人真实完成实验后的总结。
- 语气自然，不要空话套话。
