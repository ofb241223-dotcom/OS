# 截图与占位建议

本文档用于指导报告里该放哪些截图。建议在报告中先留占位符，后续再替换成自己实际运行得到的截图。

## A 类实验截图建议

- A1：`clone` 无参数运行、带参数运行、两次修改后的对比、`procterm` 同步前后
- A2：`fork_demo` 与 `exec_demo` 输出结果
- A3：消息发送与接收结果
- A4：原始程序、第一次修改、第二次修改的输出对比
- A5：虚拟内存遍历窗口的代表性区域

## B 类实验截图建议

- B1：初始资源表、安全序列、一次成功请求、一次失败请求
- B2：FIFO / LRU / CLOCK / OPT 四张结果图
- B3：FIFO / SSTF / SCAN / C-SCAN 四张结果图
- B4：读者优先运行总表
- B5：哲学家进餐过程表、统计结果
- B6：启动界面、`help`、目录切换、环境变量、后台任务、错误命令
- B7：某个时刻的进程快照、最终甘特图、汇总统计
- B8：First Fit / Best Fit / Worst Fit 三次固定序列结果
- B9：帮助界面、登录后目录、创建文件、权限修改、删除前后对比

- `[截图占位：B3 SSTF 输出结果]`

每张图下面建议补一句话，格式可以统一：

- “图 x-x 展示了……”
- “从图中可以看出……”
- “该结果说明……”
